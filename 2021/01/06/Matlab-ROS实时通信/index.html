Matlab+ROS实时通信

2021-01-06
Word count: 1.3k   |   Reading time≈ 4 min

[toc]

一、开头

MATLAB: 强大的数学软件，自带的机器人工具箱robotic system toolbox非常好用，很适合做机器人仿真，验证算法

ROS: 强大的开源机器人应用软件，提供了很多的实用工具库，感觉非常友好

很明显，如果让这两个工具强强联合肯定可以节省很多时间，实乃科研狗的福音

再也不用为了移植Matlab编写的算法到实验平台上而重写一份Python或C++代码了

但是问题来了

  1. 如何通信呢？
  2. 如何实现通信呢？？

通信的方案有很多种，但是考虑到机械臂的控制频率达到了1000hz，通信延时太高的方案都会被cut掉，只有实时通信才能确保控制算法的效果。

二、拟定方案

2.1 MATLAB中的ROS通信接口

首先登场的1号选手是MATLAB中的ROS通信接口

不得不说，matlab还是全面，直接自带了ros的通信接口

只要和ros处于同一局域网下，配置一下环境，然后输入rosinit如果连接上就OK了

接下来就完全可以同ros一样进行话题的订阅和发布了,简直完美！！

具体的操作可以参考以下文章

初始环境配置

发布话题

订阅话题

结束通信

如果有想直接看示例代码的同志可以私信我

那岂不是，本篇文章到这里就可以结束了…

qio tuo ma tie

当然不是，事情没有这么简单

当我用rostopic hz命令去查看matlab发布话题的频率时

  1. 当循环中只有一个send()函数发布消息时，频率勉强可以，基本稳定在800hz-900hz
  2. 当循环中出现receive()进行消息订阅时，频率马上下降到了100hz左右

这时候，还没加入后续代码，仅仅因为消息的订阅和发布就耗费了大量时间

额….

因为不太清楚Matlab和ROS通信的机制，所以没办法知道具体原因

官网上也有相关问题: Populating ros messages is slow

2.2 MATLAB的TCP/UDP通信

我们的2号选手就是matlab的tcp/udp通信

既然ros通信接口不好使，只能直接采用tcp或udp的通信协议

具体的通信实现可以看官网

MATLAB TCP

MATLAB UDP

首先采用的是TCP，整体传输速率没有很大提升，pass

udp虽然是不可靠的传输协议，但是我的两台电脑都在同一局域网内，应该问题不大

后面的实验也证明了udp在短距离传输下，基本甚至完全不会出现丢包误传现象

然而，matlab的udp传输速率也并没有多大提升

通过分析研究，我猜测可能的原因是Matlab的IO读写太慢了！！！

场面一度陷入了胶着

matlab通信传输这条路基本不可行了

2.3 C++版本的MATLAB Engine API

3号选手登场了

考虑到单一的matlab并不能拯救我们的通信速率

我们决定引入场外援助

首先露面的场外援助是我们的C++

新的思路是：通过C++调用matlab引擎进行算法运算，然后再通过C++的socket通信将数据发送出去，貌似就解决了问题

然而理想很丰满，现实很骨感

虽然C ++的MATLAB引擎API提供了C ++编程语言和MATLAB之间的接口

但是各项配置和使用都相当麻烦

想学习的朋友可以参考以下文章

matlab和c++混合编程—方法和步骤

Calling MATLAB from C++

但是，懒惰的我决定放弃，直接换方案

2.4 Python版本的MATLAB Engine API

都说人生苦短，我学python；人生漫漫，python是岸

起初考虑使用C++是因为项目中另一部分工作是C++实现的

思忖着统一用C++解决了也不错

但是，耐不住python的真香

思路和2.3节基本不变，只是改用了python

python调用matlab那是一个简洁明了

简简单单两句话就启动了matlab引擎

调用matlab的isprime 函数

具体教程可以见

python调用matlab API 汇总（全）

Calling MATLAB from Python

正当我以为大功告成之时，现实给我来了一记重锤

python调用matlab的引擎进行计算时，速度慢的令人发指

还不如一开始直接上matlab呢

2.4 MATLAB调用Python

痛定思痛，我发现原先思路有些许小问题

或许我不该想着其他语言来调用matlab进行计算，而是反过来，matlab调用其他语言进行通信

final思路: matlab进行算法运算，然后调用python将数据发送出去

幸运的是，matlab提供了相关支持

借助Python增强Matlab的能力，好比Matlab插上Python的翅膀，如虎添翼。

  1. 编写好python的udp通信代码

    • 只能写成函数形式

  2. 将编写好的py文件放入matlab运行路径

  3. 可直接在matlab中调用自定义的函数

参考教程如下:

知乎：Matlab调用Python

官网： MATLAB 调用 Python

到此，我们成功解决了实时通信的难题！！
